Regression Review

  • in biomedical research questions often involve understanding how biological or clinical variables relate to an outcome of interest [i.e.: treatment response]
  • substantial variability arises from a number of sources [i.e.: technical noise and unmeasured factors]
  • statistical models such as linear regression must simultaneously describe systematic effects while accounting for uncertainty
  • linear regression provides a concise framework that accomplishes this

Linear regression model Overview

Linear regression model

\[ Y=Xβ+ε,ε∼N(0,σ2) \]

  • \(Xβ\) captures the explainable relationship between predictors X and outcome Y
  • β represents effect sizes (how much Y changes when X changes)
  • explained in more detail in a later slide

Linear regression model Overview

Linear regression model

\[ Y=Xβ+ε,ε∼N(0,σ2) \]

Randomness and uncertainty

\[ ε∼N(0,σ2) \]

  • part of the model that accounts for noisiness in data
  • ε represents unobserved factors, measurement error, and randomness
  • distributional effect allows one to quantify uncertainty

Linear regression model Overview

  • Model can be written as a probability which helps one determine how likely different outcomes are

\[ Y∣X∼N(Xβ,σ2) \]

  • likelihood is a probability distribution as a function of the parameters not the data
  • measures how plausible the observed data are for different β values

\[ p(Y∣X,β, σ^2) \]

Linear regression and sampling

  • regression models how the observed outcomes Y arise from random draws from a population given the predictors X
  • Each observation is treated as a random draw from a Normal (Gaussian) distribution with mean \(X_i\)β and variance \(σ^2\)
  • Each observation is essentially a sample from a probability distribution:

\[ Yi​∣Xi​∼N(Xi​β,σ2) \]

  • β captures the systematic effect of predictors
  • \(σ^2\) represents the inherent variability

Linear regression and inference

  • inference is the process of using data from a sample to make conclusions about the underlying population or process that generated the data
  • confidence intervals (CIs) :
    • provide a range of plausible values for value being estimated [i.e.: \(\beta\)]
    • e.x.: if a model provides an estimate with a 95% CI this means within the context of sampling - if we repeated the experiment many times, 95% of the resulting intervals would contain the true we’re attempting to estimate [i.e.: \(\beta\)]
  • p-values :
    • test to compare observed values \(\hat{\beta}\) to its expected distribution under a null hypothesis [i.e.:β=0]
    • small p-value → evidence against the null hypothesis
    • large p-value → not enough evidence to reject the null hypothesis

Example: Gene Expression and regression analysis

Gene expression varies across patients even when the underlying biological effect is stable.

We model expression as a random draw around an expected value:

\[ Y_i∣X_i∼N(X_iβ_i,σ2) \]

  • \(X_iβ_i\) : expected expression given covariates
  • \(\sigma^2\) : biological + technical variability
  • likelihood

\[P(Y \mid X, \beta, \sigma^2)\]

Regression modeling questions

Once a likelihood is identified a fitted model can be used to find the following can be determined:

  1. How precise is my estimate of a drug effect?
  2. Could the observed effect be due to chance?
  3. What range of expression should I expect in a new patient?

Model comparisons

  • multiple models are often run to generate estimates

Model comparison formalizes questions like:

  • Does adding predictors meaningfully improve our explanation?
  • Which model is best supported by the observed data?

What counts as “a different model”?

Two models differ when they change any of the following:

  • Predictors included in the model
  • The distribution of the outcome (Gaussian vs Bernoulli)
  • The dependence structure (independent vs random effects)

Fit vs complexity

  • more complex models typically fit the observed data better

  • model comparison helps show whether:

    • fit improves when the model captures real structure
    • complexity grows when the model has more parameters

  • Model comparison methods reward fit and penalizes unnecessary complexity

Two common model comparison situations

In applied biomedical modeling, AIC and BIC are most often used in two recurring situations.

  • Incremental covariate adjustment (often nested models)
    You evaluate whether adding a predictor (e.g., a biomarker, batch indicator, or clinical covariate) improves model support enough to justify the added complexity.

  • Competing scientific hypotheses (often non-nested models)
    You compare alternative representations of the biology for the same outcome, such as a linear dose effect versus a nonlinear effect.

AIC and BIC compare models on a common likelihood-based scale while explicitly accounting for the number of parameters.

What AIC and BIC are optimizing

AIC and BIC combine:

  • Fit: how well the model explains the observed data (via the maximized likelihood)
  • Penalty: a cost for estimating more parameters

They answer the question:

“Did the improvement in likelihood justify the extra degrees of freedom?”

Definitions

Let:

  • \[\ell(\hat\\theta)\] be the maximized log-likelihood under the fitted model
  • \[k\] be the number of estimated parameters (including the intercept and \[\\sigma^2\] for Gaussian models)
  • \[n\] be the sample size

Then:

\[\\mathrm{AIC} = -2\\ell(\\hat\\theta) + 2k\]

\[\\mathrm{BIC} = -2\\ell(\\hat\\theta) + k\\log(n)\]

Lower is better for both criteria.

How to interpret AIC and BIC differences

AIC and BIC are most useful through differences.

For a set of candidate models, define:

\[\\Delta_i = \\mathrm{IC}_i - \\min_j \\mathrm{IC}_j\]

  • \[\\Delta = 0\] identifies the best model by that criterion
  • \[\\Delta \\in [0,2]\] indicates similar empirical support
  • \[\\Delta \\in [4,7]\] indicates meaningfully less support
  • \[\\Delta > 10\] indicates very little support relative to the best

This interpretation is applied the same way for AIC and BIC.

Background: what AIC means

AIC arises from an information-theoretic view of model quality.

AIC selects the model expected to be closest (in Kullback–Leibler divergence) to the unknown data-generating process among the candidates.

In practice, AIC tends to favor models with strong out-of-sample predictive performance, and it is comfortable choosing moderately more complex models when they improve fit.

Background: what BIC means

BIC is motivated by Bayesian model comparison.

Under common regularity conditions, BIC approximates the log Bayes factor for comparing models under a particular prior structure.

Because the penalty is \[k\\log(n)\], BIC increases the cost of complexity as sample size grows, and it tends to favor simpler models unless the added parameters improve likelihood substantially.

When AIC and BIC agree and disagree

They often agree when:

  • A real biological signal is strong and the more complex model improves likelihood substantially.

They often disagree when:

  • An added effect is modest (small improvement in likelihood), or
  • Sample size is large (BIC penalizes complexity more strongly).

In biomedical terms:

  • AIC supports models that generalize well for prediction and risk scoring.
  • BIC supports models that are strongly justified as a parsimonious scientific explanation.

Practical workflow (what you do in analysis)

  1. Fit a small set of scientifically motivated candidate models.
  2. Compute AIC and BIC for each.
  3. Compare \[\\Delta\\mathrm{AIC}\] and \[\\Delta\\mathrm{BIC}\] to identify supported models.
  4. Interpret what the selected model implies biologically (which effects are strongly supported).

Simulated biological example

We simulate a gene-expression outcome (e.g., log2 expression) measured in patients.

Scientific question:

  • Does a continuous biomarker improve explanation of expression beyond treatment status alone?
  • Is there evidence that treatment effect depends on biomarker level (an interaction)?

Simulate data (gene expression, treatment, biomarker)

set.seed(29)

n <- 120
treat <- rbinom(n, 1, 0.5)                 # 0 = control, 1 = drug
biomarker <- rnorm(n, mean = 0, sd = 1)    # standardized biomarker score

# Data-generating biology:
# - baseline expression
# - treatment shifts expression upward
# - biomarker is associated with expression
# - treatment works better when biomarker is higher (interaction)
y <- 10 +
  0.8 * treat +
  1.2 * biomarker +
  0.6 * treat * biomarker +
  rnorm(n, mean = 0, sd = 1.5)

dat <- data.frame(y, treat = factor(treat, labels = c("Control", "Drug")), biomarker)
head(dat)
          y   treat  biomarker
1  9.324421 Control -0.2590092
2 13.413836 Control  1.0352162
3  7.452651 Control -0.7519736
4  8.458286 Control  0.1145837
5 10.066736    Drug -1.4768713
6  9.903991 Control  1.3361794

Fit candidate models

m1 <- lm(y ~ treat, data = dat)
m2 <- lm(y ~ treat + biomarker, data = dat)
m3 <- lm(y ~ treat * biomarker, data = dat)  # includes main effects + interaction

Compute AIC and BIC

ic <- data.frame(
  model = c("m1: treat", "m2: treat + biomarker", "m3: treat * biomarker"),
  AIC  = c(AIC(m1), AIC(m2), AIC(m3)),
  BIC  = c(BIC(m1), BIC(m2), BIC(m3))
)

ic$DeltaAIC <- ic$AIC - min(ic$AIC)
ic$DeltaBIC <- ic$BIC - min(ic$BIC)

ic[order(ic$AIC), ]

Key results to report (AIC/BIC)

From the table:

  • The lowest AIC identifies the model with the best expected predictive support among candidates.
  • The lowest BIC identifies the most strongly supported parsimonious explanation among candidates.
  • \[\\Delta\] values quantify how much less supported the other models are relative to the best.

In a typical run of this simulation:

  • Models that include the biomarker (m2 or m3) are strongly preferred over treatment-only (m1).
  • AIC may prefer the interaction model (m3) when the interaction improves fit enough.
  • BIC may choose m2 if the interaction improvement is modest relative to its extra parameter.

Biological interpretation

When AIC and BIC both move away from treatment-only (m1):

  • The biomarker carries real signal about expression variation.
  • The treatment effect estimate is meaningfully refined by accounting for biomarker-driven heterogeneity.

When AIC prefers the interaction model (m3):

  • The data support effect modification: drug response depends on biomarker level.
  • Biologically, this matches a stratified response mechanism: patients with higher biomarker values show larger expression shifts.

When BIC prefers the additive model (m2):

  • The biomarker is supported as a main effect.
  • Evidence for interaction is weaker relative to the complexity cost, so the most defensible explanation is parallel drug effect across biomarker levels.

A compact reporting sentence (what goes in a results paragraph)

“We compared candidate linear models using AIC and BIC. Models including the biomarker were strongly supported relative to the treatment-only model (large \[\\Delta\\mathrm{AIC}\] and \[\\Delta\\mathrm{BIC}\]). AIC favored the interaction model when treatment response varied by biomarker level, whereas BIC more often selected the additive model, indicating that interaction evidence was weaker relative to the complexity penalty at this sample size.”

Takeaway for practice

AIC and BIC provide fast, reproducible model comparison that aligns with scientific decision-making:

  • Use AIC when your goal is stable predictive performance across new patients.
  • Use BIC when your goal is a concise biological explanation with strong support.

In both cases, the biology is interpreted through the selected terms: main effects indicate additive contributions; interactions indicate effect modification and stratified response patterns.
